# 🌉 Alpaca Bridge Implementation (seestar.service)

> **Objective:** Mandates the communication protocol, service verification, and routing for the Seestar ALP bridge.
> **Version:** 1.4.17 (Infrastructure Baseline)

## ⚙️ The Local Service Reality
The Seestar bridge does NOT run under our virtual environment. It is an independent system service.
* **Service Name:** `seestar.service` (Managed by systemd, PPID 1).
* **Execution:** Runs via a globally installed pyenv (`~/.pyenv/versions/ssc-3.13.5/bin/python3`).
* **Dependency:** All internal code MUST verify `systemctl is-active seestar.service` before making API calls.

## 📡 The ET Protocol (Communication)
* **API Port:** 5555.
* **Routing:** Always query `http://127.0.0.1:5555` to remain immune to network drops or AP Fallback mode switches.
* **Endpoints:** Never hardcode `/telescope/0/`. Endpoints must be dynamically read from `data/fleet_schema.json` generated by Block 2.
* **Actions:** All `Action` calls MUST use the **PUT** method and include `ClientID` and `ClientTransactionID`.
